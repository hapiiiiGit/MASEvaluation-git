# Video Verification Claims Web App PRD

## 1. Language & Project Info
- **Language**: English
- **Programming Language**: Vite, React, MUI, Tailwind CSS
- **Project Name**: video_verification_claims_web_app

## 2. Product Definition
### Product Goals
1. Implement a secure and user-friendly claims submission process with Google authentication.
2. Provide a comprehensive claims dashboard for users to track their claims status.

### User Stories
1. As a user, I want to log in using my Google account so that I can securely access my claims.
2. As a claims adjuster, I want to view all submitted claims in a dashboard so that I can manage and verify them efficiently.
3. As a user, I want to initiate a video call for claims verification so that I can provide real-time evidence.
4. As a user, I want to fill out a multi-step claim form so that I can provide detailed information about my claim.
5. As a user, I want to receive email notifications about my claim status so that I am always informed.

### Competitive Analysis
1. **Product A**
   - **Pros**: User-friendly interface, robust security features.
   - **Cons**: Limited customization options.
2. **Product B**
   - **Pros**: Comprehensive dashboard, excellent customer support.
   - **Cons**: High pricing model.
3. **Product C**
   - **Pros**: Integrates well with third-party services, good performance.
   - **Cons**: Steep learning curve for new users.
4. **Product D**
   - **Pros**: Strong video conferencing capabilities, mobile-friendly.
   - **Cons**: Occasional bugs in the system.
5. **Product E**
   - **Pros**: Affordable pricing, easy to set up.
   - **Cons**: Lacks advanced features.

### Technical Specifications
#### Requirements Analysis
- **Google Authentication**: Implement OAuth 2.0 for secure user login.
- **Claims Dashboard**: A user-friendly interface to view and manage claims.
- **Video Calls**: Integrate Jitsi Meet for real-time video communication.
- **Geo-location Capture**: Use HTML5 Geolocation API to capture user location.
- **Media Storage**: Utilize AWS S3 for storing media files securely.
- **Multi-step Claim Forms**: Design forms that guide users through the claims process.
- **PDF Report Generation**: Generate downloadable PDF reports for claims.
- **Email Notifications**: Use FastAPI or AWS SES for sending email updates.

#### Requirements Pool
- **P0 (Must-have)**:
  - Google Authentication
  - Claims Dashboard
  - Video Calls
  - Multi-step Claim Forms
- **P1 (Should-have)**:
  - Geo-location Capture
  - Media Storage in AWS S3
  - PDF Report Generation
- **P2 (Nice-to-have)**:
  - Email Notifications

#### UI Design Draft
- **Login Page**: A simple interface for Google authentication with a prominent login button.
- **Claims Dashboard**: A grid layout displaying all claims with status indicators and action buttons for each claim.
- **Video Call Interface**: A full-screen layout for Jitsi Meet with options to mute, end call, and share screen.
- **Claim Form**: A multi-step form with progress indicators, input fields for user information, and file upload options.
- **PDF Report Page**: A layout for displaying generated reports with download options.

#### Open Questions
1. What specific user data will be required for the claims process?
2. How will we handle user data privacy and compliance with regulations?
3. What are the performance expectations for video calls during peak usage?
